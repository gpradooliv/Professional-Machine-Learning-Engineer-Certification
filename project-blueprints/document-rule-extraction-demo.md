# Blueprint — Document Rule Extraction Demo

**Objetivo:** demonstrar ingestão versionada, extração híbrida, normalização, proveniência, diff e human-in-the-loop.

**Dataset seguro:** PDFs sintéticos próprios ou documentos públicos com licença compatível.

**Stack:** Python, OCR ou Document AI, LLM estruturado, PostgreSQL ou BigQuery.

**Métricas:** precision, recall e F1 por campo; conflitos detectados; qualidade do diff; taxa de revisão humana.

**Regra de PI:** não publicar manuais internos, documentos de processo seletivo, regras comerciais reais ou prints.
